## Terraform Structure Overhaul + humboldt VPC

Restructured the Infrastructure repo layout. Old pattern had a separate state file per resource type (vpc/, iam/, etc.) ‚Äî too many `terraform init` runs, hard to reference resources across files.

New pattern: **single state per cluster per provider.**

```
platform/humboldt/
‚îú‚îÄ‚îÄ config.yaml
‚îî‚îÄ‚îÄ aws/
    ‚îú‚îÄ‚îÄ providers.tf   ‚Üê backend + provider + local.config + local.default_tags
    ‚îú‚îÄ‚îÄ vpc.tf
    ‚îú‚îÄ‚îÄ ec2.tf
    ‚îî‚îÄ‚îÄ outputs.tf
```

`local.config` always lives in `providers.tf` ‚Äî shared across every file in the state automatically. Never in a separate `locals.tf`.

Updated `terraform-module` skill to reflect this pattern.

## Tagging Policy

Owner's rule: **never create untagged resources.** Standard approach going forward:

- AWS provider `default_tags` block in `providers.tf` ‚Äî every resource gets `ManagedBy`, `Account`, `Network`, `Cluster`, `Environment` automatically
- For providers without `default_tags` (Grafana, Tailscale etc.), pass `local.default_tags` explicitly
- Individual resources only need resource-specific tags (e.g. `Name`)

Applied to `platform/humboldt/aws/providers.tf` immediately.

Created `platform/humboldt/aws/vpc.tf` ‚Äî single NAT gateway, nodes in private subnets, single AZ (us-east-1a). Intentionally not fault tolerant to avoid cross-AZ data transfer costs.

## EC2 Module Overhaul

Rebuilt `modules/aws/ec2` to be fully self-contained ‚Äî one module call creates everything:

- **Security group** bundled inside the module via dynamic ingress/egress blocks. Default egress allows all outbound. Caller passes `ingress_rules` list or leaves it empty (Tailscale needs no ingress).
- **IAM role + instance profile** auto-created when `policy_arns` is set (same pattern as before).
- Removed `security_group_ids` as a required input ‚Äî replaced with `vpc_id` (for SG creation) and optional `extra_security_group_ids`.

- `secrets.tf` ‚Äî Tailscale auth key secret + scoped IAM policy
- `templates/tailscale.sh` ‚Äî user data script, rendered via `templatefile()`

## Secrets Manager Module

Created `modules/aws/secrets-manager`:

- Secret with configurable recovery window (0 = force delete, 7‚Äì30 = standard)
- AWS-managed KMS by default; pass `kms_key_arn` for CMK
- Scoped read policy: `GetSecretValue`, `DescribeSecret` (+ `kms:Decrypt` if CMK)
- Scoped write policy: `PutSecretValue`, `UpdateSecret`, `DeleteSecret` (+ `kms:GenerateDataKey` if CMK)
- Outputs: `arn`, `name`, `read_policy_arn`, `write_policy_arn`
- Caller passes tags from their `config.yaml` ‚Äî module only sets `Name`

Lambda-based secret version updater noted as a later project.

## Tailscale Terraform Integration

Replaced bash-based auth key management with the Tailscale Terraform provider. Separate state per provider ‚Äî aws things with aws, tailscale things with tailscale.

`platform/humboldt/tailscale/` (new state):
- `tailscale_tailnet_key` ‚Äî generates auth key automatically on apply (preauthorized, non-ephemeral, 90-day expiry, `tag:subnet-router`)

`platform/humboldt/aws/` ‚Äî no longer manages the SSM parameter, just reads its ARN via data source for the inline IAM policy. EC2 fetches the actual value at boot.

Apply order: tailscale state first, then aws state.
